4. Regras de Negócio Implementadas
  • RN01 — Usuário inativo: Não é permitido abrir chamado para usuário com ativo = 0. Verificado no DAO antes do INSERT.
  • RN02 — Chamado fechado não reabre: Ao tentar alterar o status de um chamado FECHADO para qualquer outro, o sistema lança exceção.
  • RN03 — Avanço automático de status: Ao atribuir o primeiro técnico a um chamado ABERTO, o status avança automaticamente para EM_ATENDIMENTO.
  • RN04 — Integridade referencial: Usuário com chamados vinculados não pode ser excluído (verificado no DAO antes do DELETE).
  • RN05 — Data de fechamento automática: Ao resolver ou fechar um chamado, dt_fechamento é preenchida automaticamente com o timestamp atual.
